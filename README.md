# GigaChat AI Assistant

Android-приложение: чаты с AI на базе **GigaChat API**, локальное хранение истории, вход через **Firebase Authentication**.


## Стек

- **Kotlin**, **Jetpack Compose**, Material 3  
- **Navigation Compose** (type-safe routes)  
- **Firebase Authentication** (+ **Firebase Storage** для аватара)  
- **Room** — чаты и сообщения на устройстве  
- **Paging 3** — список чатов с подгрузкой порциями  
- **GigaChat** — OAuth и REST-запросы к чату (токен и ключ в `local.properties`)  
- **DataStore** (через `SettingsRepository`) — тема приложения  

## Экран входа и регистрация

- Вход по **email и паролю** через Firebase; при успехе — переход к списку чатов.  
- Отдельный экран **регистрации** нового пользователя.  
- **Валидация** email и пароля до отправки; сообщения об ошибках Firebase через строки ресурсов.  
- Состояние загрузки: кнопка блокируется, показывается индикатор.  
- **Автовход**: при старте, если `FirebaseAuth` уже содержит пользователя, открывается не логин, а список чатов.  
- **Вход через Google** — при наличии `firebase.web.client.id` в `local.properties` (Credential Manager / Google ID); иначе кнопка не показывается.  
- **Анимация** появления блока полей входа (slide + fade).  
- Светлая и тёмная тема (см. профиль и `Theme.kt`).  

## Список чатов

- **App bar** с меню (drawer) и кнопкой поиска.  
- **Строка поиска** и кнопка применения поиска; фильтрация по названию чата через Room (`LIKE`), с **debounce** ввода.  
- **FAB** — создание нового чата и переход в экран чата.  
- Элементы списка: иконка, название, дата/время; **swipe** для удаления чата.  
- **Пагинация** списка чатов (Paging + `PagingSource`).  
- Сортировка по **последней активности** (`lastActivityAt`).  
- Действие **очистить всю историю чатов** (подтверждение в bottom sheet).  
- Состояния загрузки и ошибок (в т.ч. при догрузке в конце списка — retry).  

## Экран чата

- Верхняя панель: **назад**, заголовок чата.  
- **Лента сообщений** пользователя и ассистента; история из Room, обновление через Flow.  
- **Многострочное поле ввода** и отправка; при отправке — индикатор на месте кнопки.  
- Запрос к **GigaChat** по накопленной истории сообщений; ответ ассистента сохраняется в БД.  
- При ошибке сети/API — сообщение об ошибке и **повтор** (retry).  
- **Поделиться** текстом ответа ассистента через системный `ACTION_SEND`.  
- Автопереименование чата с дефолтного названия после первого ответа ассистента (по логике в `ChatViewModel`).  

## Профиль

- Данные пользователя из **Firebase** (отображение имени, email и т.д.).  
- **Редактирование имени** (сохранение в профиль Firebase).  
- **Аватар**: выбор изображения из галереи, загрузка в **Firebase Storage**, обновление `photoUrl` в профиле; отображение через **Coil**.  
- Блок **квоты GigaChat** (баланс по API) с состояниями загрузки / ошибки / повтор.  
- Переключение **темы**: системная / светлая / тёмная.  
- **Выход**: sign out Firebase, очистка локальной БД Room, переход на экран входа.  

## Навигация (drawer)

Боковое **modal drawer** с пунктами: новый чат, список чатов, изображения, профиль.  
«Новый чат» создаёт чат в Room и открывает экран переписки.

## Экран «Изображения»

Заглушка с текстом-плейсхолдером: **генерация картинок в приложении не реализована**.

## Ошибки и загрузка

На основных экранах предусмотрены индикаторы загрузки и отображение ошибок пользователю (включая сетевые сценарии там, где они обрабатываются в репозитории).

## Сборка и секреты

1. Подключить **Firebase**: файл `google-services.json` в модуль `app` (как обычно для Android).  
2. В **`local.properties`** (не коммитить полные секреты в открытый репозиторий):  
   - `gigachat.auth.key` — ключ авторизации GigaChat для `BuildConfig`.  
   - `firebase.web.client.id` — Web Client ID из Firebase Console для **Sign-In with Google** (если пусто — кнопка Google на экране входа скрыта).  

Сборка: из корня проекта `./gradlew :app:assembleDebug` (или через Android Studio).
